# Build Request Intake

Complete this once to start an autonomous ZeroHandoff run. This is the initial
request, not a generated delivery artifact. SENSE converts it into the first
artifact—the Opportunity Model—and the pipeline creates everything after that.

## Required input

### 1. What do you want built?

Describe the product idea in a few plain-language sentences.

> 

### 2. Who is it for?

Identify the primary user or audience.

> 

### 3. What outcome should it create?

Describe what should become easier, faster, safer, or possible for that user.

> 

### 4. What constraints are non-negotiable?

List any required platform, technology, integrations, accessibility needs,
security or compliance rules, deadline, budget, or hard exclusions. Write
“None” when there are no fixed constraints.

## Autonomous default

Anything left blank is intentionally delegated to ZeroHandoff. The pipeline
selects a reasonable default, records the assumption in its generated artifacts
and JSON run log, and continues without requesting another human decision. If it
cannot proceed safely, it stops and records the blocker in the run log.
