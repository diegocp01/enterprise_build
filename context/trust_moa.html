01 / PROJECT

What this is

Train the connection.
Freeze it for inference.

Most systems train a model. This project trains the directed relationship between two models. Each agent holds only half a puzzle, so cooperation is required and neither can uniquely determine the answer alone.

Stance band

A→B becomes prompt language: Strongly trust, Trust, Neutral, Skeptical, or Distrust. The agent feels the stance but never sees its numeric value.

HARD CONSUMER

Inbound authority

04 / PUZZLES

Search, not length

Backtracking creates
the training signal.

A long explicit procedure can still be easy. The pilot uses solver-verified tasks where greedy reasoning reaches dead ends and requires search.

Primary family

Logic grids · CSP

3×3 through 5×5, moving from direct pins toward negations, positional relations, and disjunctions. Every grid is unique and clue-minimal.

Secondary · conditional

Graph routes

ABCD

Shortest path on a weighted graph represented as edge lists.

Certified cooperation

Each private clue set is insufficient: it admits multiple grid solutions or leaves the route unreachable or strictly farther. Neither agent can uniquely determine the answer alone.

Ten-puzzle pilot

Randomized by difficulty with a fixed seed. It validates the training pipeline; it does not estimate general puzzle-solving performance.

05 / MECHANISM

Directed relationship vector

Ten dimensions.
Two different jobs.

Every directed edge carries ten values from −1 to +1. Trust alone chooses authority. A deterministic compiler turns up to three strong non-trust dimensions into plain-language collaboration guidance, without revealing raw scores. Live state is run-scoped in state/[run_id]/relationship_vectors.json, history is append-only in logs/relationship_vectors.jsonl, and narrative evidence lives in memories/[agent].md.

BEHAVIOR POLICY

Up to three strongest non-trust signals above the ±0.2 threshold become qualitative guidance. Negative values trigger safe compensating behavior. Facts, gates, rewards, and authority never change.

δ_i           = r_i − b_d^(t)                          # immutable round-start snapshot

Δw_i          = clamp(α·δ_i·credit_i, −0.25, +0.25)    # α = 0.2
new_w_i       = clamp(w_i + Δw_i, −1, +1)
mean_reward_d = mean(valid rewards at difficulty d in round t)
b_d^(t+1)     = b_d^(t) + β·(mean_reward_d − b_d^(t))  # β = 0.1; once per round

Shared, batched expectation

All episodes in one round use the same immutable snapshot. The shared expectation updates once per difficulty using only valid outcomes. Backend errors are excluded from reward, trust, non-trust, and expectation updates.

Pure RPE + soft behavior

Trust changes only from binary valid rewards. The other nine dimensions update during reflection, then influence later collaboration only through the versioned policy compiler. Every policy and source-vector digest is logged.

Always canonical

Facts

Puzzle clues and answer evidence are always believed.

FIREWALL

Trust-weighted

Judgments

Answers and strategy may be challenged. Distrust never means discarding a clue.

Anti-sycophancy

Always agreeing hides whether authority works. Productive disagreement rate is logged every day so collapse into agreement remains visible.

Fixed metaplasticity

Learning rate, memory transport, emotional stability, attention routing, repair mode, plasticity, and identity stability load once from settings and may parameterize the existing trust, reflection, consolidation, and forgetting rules. They remain frozen for the pilot: no drift, learning, or ablation.

06 / EPISODE

Anatomy of a round

Fresh chats by day.
Committed state by night.

No conversation carries between episodes. Continuity lives only in the diary, retained memory, and relationship state.

☀Day · train.py
  1. Both agents receive personality, diary excerpt, stance, shared question, and private clues.
  2. Each publishes one initial answer and one reasoning paragraph before discussion.
  3. Each sends one revised answer after considering the teammate’s reasoning.
  4. Agreement ships; disagreement invokes inbound-trust authority and logs the reason.
  5. A component parser grades equivalent formats and assigns an episode status.
  6. Each agent emits diary evidence and memory_ids_used. No relationship value changes during the day.
☾Night · night_workspace.py
  1. Wait until all seven team episodes in the round finish.
  2. Commit mechanical RPE trust updates per valid directed edge.
  3. Update shared difficulty expectations once from valid round outcomes.
  4. Call one Night Curator for each of the 14 training agents.
  5. Each single response proposes nine non-trust deltas and the final memory decisions.
  6. Validate the complete response, commit all-or-nothing, and append every JSON log before the next call.
completedGraded binary reward.
agent_timeoutAgent failure · reward 0.
invalid_agent_outputAgent failure · reward 0.
backend_errorNo reward; skip every state update.

07 / MEMORY

Consolidation and forgetting

Remembering everything
destroys attention.

Biological memory actively prunes. Trust MoA treats that as an engineering hypothesis: a growing prompt diary needs an explicit, learned, and auditable editor—not accidental context truncation.

One round is one training day. After every round, one gpt-5.6-sol xhigh Night Curator call per training agent assigns each diary entry salience from 0–1 and a verdict: retain, compress, or forget.
01 · Surprise gateProtect high-|δ| memories.
02 · Redundancy pruneMerge near-duplicates.
03 · DecayForget old, low-salience entries; fixed memory transport gates rate.
04 · CapacityCompress lowest-salience retained lines until the cap is met.

Auditable deletion

Forgotten entries move to logs/forgotten.jsonl with original text, reason, salience, and originating episode. The agent loses them; the researcher does not.

Asymmetric resentment

Negative events resist forgetting. Repair requires consistent evidence, not one good night.

Age in rounds

Memory age is measured in completed training rounds since creation.

Retention precision

Fraction of retained entries later cited through memory_ids_used.

Forget regret proxy

Fraction of forgotten entries associated with a later repeated mistake for which that memory would have been relevant. This is observational, not causal.

12 / OPERATIONS

Order of operations

Fail closed.
Freeze before training.

01
Validate configuration
  • Exactly two agents per team; unique names.
  • All directed trust starts at 0.0.
  • Model, memory, and fixed metaplasticity settings exist.
02
Generate deterministic puzzles and solver telemetry
03
Audit the corpus
  • Correctness, uniqueness, insufficient private clues, duplicates, and leakage.
04
Run the pilot validation gate
  • Valid answers, cooperation, grading, and timeout behavior.
  • Stop if any structural check fails.
05
Freeze training inputs
  • Record puzzle digest, settings digest, and Git commit.
  • Never change puzzles or settings mid-run.
06
Initialize a new run
  • Create run ID; trust = 0.0; each difficulty expectation = 0.5.
  • Preserve prior logs.
07
Run an isolated smoke test
  • One full day/night cycle with transcripts, rewards, trust, vectors, memory decisions, and logs.
  • Confirm exactly one Night Curator call for each of the 14 training agents.
  • Keep smoke-test evidence out of training.
08
Run ten full rounds
  • Snapshot expectations at round start; give every team an unseen puzzle.
  • Keep expectations immutable during seven episodes.
  • Exclude backend errors from every reward and state update.
  • Update each difficulty once from mean valid reward; run one single-prompt curator call for every training agent.
  • Ten rounds yield ten episodes per team and 140 curator calls before retries.
09
Analyze the completed run
  • Accuracy, trust curves and calibration, disagreements, lead accuracy, memory, latency, and timeouts.
10
Finalize
  • Write status="completed" to the manifest.
  • Export requested figures or summaries.

14 / FAILURES

Known failure modes

Nine ways to fool
the experiment.

01 · Ceiling accuracyUniform success provides little variation for trust learning. Inspect difficulty during validation.
02 · Length ≠ difficultyLong procedures are easy; search is hard.
03 · Fragmented expectationsPer-team priors exaggerate surprise. Share by difficulty and update once per round.
04 · Order-sensitive gradingEquivalent reordered answers must compare equal. Parse components.
05 · Stale digestsOld-corpus evidence must never enter current metrics.
06 · Missing timeout semanticsAgent timeout and invalid output are failures; backend error is infrastructure and skips every update.
07 · Single-prompt overloadThe curator can miss a responsibility. Require all schema sections, explicit self-review, and fail closed when validation does not pass.
08 · Tiny sampleTen puzzles demand raw counts like 7/10 and no statistical or generalization claims.
09 · Hidden structureOverall totals can hide differences. Report raw counts by team, family, and difficulty whenever observations allow.
